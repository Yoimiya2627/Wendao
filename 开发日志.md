## v26：美术全面优化（已完成）

### 1. 水墨 Shader 增强（shaders/ink_wash.gdshader）
- 新增纸纹噪声（`grain_strength=0.025`）：像素级颗粒感模拟宣纸纤维
- 新增横纹扫描线（`scanline_count=120`，`scanline_strength=0.03`）：旧画卷横向纤维纹路
- 三项参数均可在 InkWashOverlay ShaderMaterial 属性面板单独调节

### 2. 战斗粒子效果（scripts/BattleParticles.gd，scenes/BattleScene.tscn，scripts/BattleUI.gd）
- 新建 `BattleParticles.gd`：管理两种 CPUParticles2D，挂载在 CanvasLayer layer=6（高于 InkWashOverlay）
  - 命中墨点飞溅：深色粒子，高爆发性，0.55s 生命周期
  - 技能金色爆裂：金色粒子，蓄势/感应时触发，0.8s 生命周期
- BattleScene.tscn 新增 BattleParticles 节点
- BattleUI.gd `_execute_action`：敌方/玩家受击调用 `play_hit()`，CHARGE/SENSE 调用 `play_skill()`，位置取对应面板 `get_global_rect().get_center()`

### 3. 对话框美化（scripts/DialogueBox.gd）
- `_apply_styles()` 新增顶部金色分割线（2px ColorRect，左右各留18px）
- `_add_corner_ornaments()`：四角添加 ◆ 金色装饰标签，带 `_corner_root` 判重保护防重复创建
- 新增人物剪影立绘面板（见第7条），所有新增元素 `mouse_filter=IGNORE` 不干扰点击

### 4. 场景环境粒子（scripts/ShopScene.gd，scripts/TempleScene.gd）
- ShopScene：`_spawn_ambient_particles()` 在 CanvasLayer layer=1 生成暖色尘埃粒子（金黄半透明，缓慢漂浮，25粒/帧）
- TempleScene：同样结构，粒子改为淡紫灰色、数量更少（15粒）、速度更慢，营造废庙幽森氛围
- 两处均在 `_ready()` 末尾调用，layer=1 确保在水墨遮罩(layer=5)下方

### 5. 按钮动画 AutoLoad（scripts/ButtonAnimator.gd，project.godot）
- 新建 `ButtonAnimator.gd` AutoLoad：通过 `get_tree().node_added` 信号自动为场景中所有 Button 绑定 Tween 动画
  - 悬停：scale 1.04x（0.12s，TRANS_BACK）
  - 按下：scale 0.94x（0.07s）
  - 松开：弹回 scale 1.0（TRANS_BACK 弹跳）
- `_sync_pivot()` 在每个事件中同步设置 `pivot_offset = size * 0.5`，确保从按钮中心展开
- `project.godot` 注册 ButtonAnimator 为 AutoLoad

### 6. TileMap 视觉增强（scripts/TownScene.gd）
- `_fill_tile()` 新增像素级随机色彩扰动（±0.035），模拟手绘水墨质感，每格不再完全平铺
- `_tile_map.z_index = 0` 显式赋值，避免与 WorldDecorations(z_index=2) 层级歧义
- 新增 `_spawn_world_decorations()`：在格坐标匹配位置生成三处 Polygon2D 装饰物
  - **老榕树**（格17,14）：9点多边形树冠 + 树干，与 BUBBLE_NODES "老榕树的根" 位置对应
  - **古井**（格22,17）：外圈八边形 + 黑色井口，与 BUBBLE_NODES "井沿磨得很光" 位置对应
  - **公告栏**（格22,13）：支柱 + 牌面 + 两条横纹装饰线，与 HIDDEN_INTERACT_NODES notice_board 位置对应

### 7. 角色剪影立绘（scripts/PortraitControl.gd，scripts/DialogueBox.gd）
- 新建 `PortraitControl.gd`：继承 Control，通过 `_draw()` 程序化绘制半身剪影，无需外部图片

## v40.1：再修 4 条中等级 BUG

继 v40 之后，继续清掉报告里的 4 条代表性欠账：

### 修复 #4：DialogueBox IoC 悬空指针
- `DialogueBox._exit_tree`：若 `DialogueManager._dialogue_box == self`，主动清为 null，防止旧节点销毁后指针悬空
- `DialogueManager._get_dialogue_box`：加 `is_instance_valid` 过滤，顺手清掉悬空引用

**修复动机**：场景切换瞬间（旧 DialogueBox 已 `queue_free` 但新场景 `_ready` 未执行），`force_stop` 后若立刻有代码触发对话，会访问已释放节点。

### 修复 #6：BOSS 立绘一帧混叠（BattleUI.gd + EnemySilhouettes.gd）
- `BattleUI._on_boss_phase2_started`：清理旧立绘时，在 `queue_free` 前先 `remove_child`，同步脱离渲染
- `EnemySilhouettes.clear`：同样改为 `remove_child` + `queue_free`（公共工具一起修）

**修复动机**：`queue_free` 是异步的（到帧末才真删），原代码新 `_draw_block` 立刻 `add_child`，旧立绘多活一帧，产生"P1/P2 立绘叠加"的视觉瑕疵。

### 修复 #8：存档读取类型防御（GameData.gd）
- 新增 `_safe_typed_array(raw, elem_type, fallback)`：逐元素校验类型，不匹配的丢弃而非整体抛错
- 替换 `triggered_events / stones_read / unlocked_old_items / charm_spoken_stages` 四处的 `Array(data.get(...), TYPE_X, ...)` 为安全转换
- 额外兼容：旧存档 bool 存成 0/1、StringName 存成 String 都能被吸收

**修复动机**：损坏/篡改存档原本会让 `Array(...)` 直接抛 runtime error，玩家卡在主菜单无法读档。

### 修复 #9：NPC.required_phase 动态响应（NPC.gd）
- `_ready` 抽出 `_refresh_phase_visibility()` 并订阅 `GameData.story_phase_changed`
- `_exit_tree` 断开订阅
- `_on_story_phase_changed` / `_refresh_phase_visibility` 都保留 `_gone` 保护位，防止已 disappear 的 NPC 被错误复活

**修复动机**：`set_phase(3)` 等同场景内 phase 跃迁（测灵石对话后）不再要求离开-回来刷新，NPC 按 phase 即时 show/hide。

### 核查遗漏
`TownScene` 有自己的"强制矫正测验师/老婆婆"逻辑（`_ready` 里基于 `story_phase` 直调 `examiner.dialogue_scene_id = "..."`），不会与新的 `_refresh_phase_visibility` 冲突——前者改对话 ID，后者只控可见性。

## v40.2：BOSS 觉醒演出仪式化重编

### 动机
真人玩家（回合制偏好）10 分钟速通 + "读完了但没记住"的反馈，暴露了一个更深问题：
**剥离文字之后，游戏的非文字记忆锚太少**。觉醒一击（"——挥出去"）是本作唯一有仪式感潜质
的峰值瞬间，但原版演出约 2 秒就结束，量级与"普通 UI 点击反馈"相当，
**达不到"玩家十年后还记得"的密度**。

### 新五幕结构
| 幕 | 时长 | 视听内容 |
|---|---|---|
| 1 · 静默 | 3.5s | UI 淡出 1.5s + BGM 同步 `fade_bgm_to(0, 1.5)` + 纯静默 2s |
| 2 · 浮现 | 3.5s | 觉醒按钮淡入 2s + 呼吸动画 + 1.5s 停留 |
| 3 · 决定 | 玩家决定 + 0.7s | 按下 → 按钮淡出 0.3s → 静默 0.4s → 爆发 |
| 4 · 爆发 | 2.3s | 重震屏 (1.8×, 15 次) + 长白光 (峰 1.0, 保 0.6, 退 1.5) + 剑穗爆发 + 中央浮字"混沌灵根，觉醒" |
| 5 · 余波 | 1.5s | `awakening` BGM 2s 淡入 + 无 UI 静止 → 切到 `boss_awakening` 对话 |

**固定演出总时长：约 10-11 秒**（原版 ~2 秒），4-5 倍提升。设计要点是**"静默比爆发更有力"**——
两段纯空白（2s + 0.4s）是让玩家的眼睛和耳朵停下来。

### 改动文件
- `BattleEffects.gd`
  - `shake_screen` 加 `(intensity=1.0, shakes=6, per_shake=0.04)` 参数，保留默认值不破坏原调用
  - `flash_white` 加 `(peak=0.9, rise=0.12, hold=0.3, fade=0.8)` 参数，同上
- `BattleUI.gd`
  - `_on_ready_for_awakening` 重写为第 1-3 幕（UI+BGM 1.5s 慢淡出、2s 纯静默、按钮 2s 慢淡入+呼吸、1.5s 停留后才可点）
  - `_on_awakening_triggered` 重写为第 4-5 幕（重震屏+长白光+浮字+BGM 淡入+1.5s 余波）
  - 新增 `_spawn_center_awakening_text`：屏幕中央"混沌灵根，觉醒"大号浮字（44pt 暖金色+黑描边），0.8s 淡入 + 1.2s 保持 + 1.0s 淡出

### 非代码收益
**这是给扫读玩家（不读字也能记住）的视觉锚**。对深度玩家不折损（依旧能读 log），
对扫读玩家等于新建一个"截图时刻"。

### 待验证
找 1-2 位试玩者打到 BOSS 这一段，观察是否"愣一下 / 手停一下 / 愿意截图"——
若无反应，说明节奏还不够或振幅还不够，可继续调参（所有参数都已外露）。
