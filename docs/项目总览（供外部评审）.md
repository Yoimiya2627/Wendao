## 三、架构设计

### AutoLoad 单例（5个）
- **GameData** — 全局状态、存档读档、玩家属性、story_phase 状态机
- **DialogueManager** — 数据驱动对话系统，读取 chapter1.json
- **SceneTransition** — 场景切换（淡入淡出）
- **UIManager** — 常驻 HUD（HP/心绪面板/背包/功法栏/ESC菜单）
- **VirtualJoystick** — 虚拟摇杆（PC 端自动隐藏）

### 数据驱动设计
- 全部对话存储在 `data/chapter1.json`，不在代码中硬编码
- 对话节点支持：文本显示、选项分支、事件触发、自动跳转
- story_phase（0-5）驱动全局叙事流转，各场景根据 phase 切换 NPC 状态

### 战斗系统架构
- `BattleManager`（extends RefCounted）— 纯逻辑，不挂场景树
- `BattleUI`（extends Control）— UI 层，接收信号驱动演出
- 信号驱动：`turn_changed`、`battle_log`、`battle_ended`、`boss_phase2_started`、`ready_for_awakening`、`awakening_triggered`
- 同回合结算制：玩家先选择行动 → 按先手顺序结算双方 → 日志一次性展示

### 存档系统
- 路径：`user://save.json`
- 安全写入：先写 `.tmp` 再替换
- 触发时机：出门、离开茶馆、进废庙前
- 战斗中禁止存档

## 四、游戏内容

### 世界观
- **九洲仙墟**：修仙界以灵根品阶决定命运，一品最强九品最弱
- **苏云晚**：16岁女主，测灵显示"无灵根"，实为混沌灵根（万年未现，天道的BUG）
- **苏明（爹）**：表面普通凡人，实为隐藏大佬，深爱女儿，第一章全程伏笔
- **专属境界体系**：蒙昧→藏锋→观律→破执→问道→归墟（混沌灵根独有）

### 第一章流程（5条主线链路，全部已实现）

| 链路 | 内容 | 预估时间 |
|------|------|---------|
| 链路一 | 杂货铺清晨：猫咪互动、大鱼叼来旧剑穗、出门准备 | 5-8分钟 |
| 链路二 | 测灵广场：NPC对话、灵根测试、测出"无灵根" | 5-8分钟 |
| 链路三 | 回家流程：回程旁白、回家猫咪安慰、发现爹的便条 | 5-8分钟 |
| 链路四 | 废庙流程：五房间探索、碑文阅读、三种妖兽战斗、BOSS觉醒 | 20-30分钟 |
| 链路五 | 章末流程：路径A（一起走）/ 路径B（先回去看爹） | 3-5分钟 |
| **合计** | | **51-79分钟** |

### 战斗系统

**三技能：**
| 技能 | 效果 | 解锁 |
|------|------|------|
| 蓄势 | 跳过本回合，下一击×2且无视防御 | 初始 |
| 淬血 | 自损10HP，造成 ATK×1.5+已损失HP×0.5 真实伤害 | 初始 |
| 感应 | 本回合减伤50%，下一击无视防御×1.2，预告敌方行动 | 读碑文后 |

**三种妖兽+BOSS：**
- 幽影狼（HP60/ATK10/DEF2）— 蓄力预警机制
- 石皮蟾（HP100/ATK14/DEF5）— 硬壳防御/毒液/前摇预警
- 虚形魇 BOSS（HP120/ATK16/DEF4）— 两阶段：虚影爪+模糊+侵蚀 → 锁血+觉醒一击

**回合制设计：**
- 同回合结算：玩家先选择行动，再按先手顺序结算
- 敌方先手（幽影狼/虚形魇）= 敌先结算再玩家结算
- 玩家先手（石皮蟾）= 玩家先结算再敌结算
- 死方不再行动
- Buff 跨回合保留（蓄势/感应增益直到攻击才消耗）

**消耗品系统：**
- 伤药（30HP/10灵石）、定神香（战斗开局自动+感应buff/20灵石）、辟邪符（免疫毒/15灵石）
- 灵石来源：击败妖兽（狼8/蟾18）

**战斗演出效果：**
- 受伤面板闪红
- 重击屏幕震动（≥15点伤害触发）
- HP条平滑缓动过渡
- 觉醒全屏白光+震动

### 隐藏探索内容（8项）
- 传音阵、三百六十五道划痕（有内在联系）
- 残页×4（散布在大地图和废庙）
- 旧剑穗（大鱼互动后自动给予）
- 算命先生铜钱（隐藏NPC）

### 章末设计
- 路径A（一起走）："她只带了那张平安符，还有腰间装了半壶水的葫芦。走出废庙的时候，她没有回头。"
- 路径B（先回去看爹）："一封没有署名的信。一碗早就凉透的饭。"
- 悬念钩子：「有人动了手脚。那个人，认识你。」

## 五、技术亮点

1. **数据驱动对话系统** — 85个场景全在 JSON 中，支持分支/事件/跳转，不硬编码
2. **story_phase 状态机** — 单一变量驱动全局叙事流转，各场景根据 phase 差分 NPC 行为
3. **同回合结算制** — 解决传统回合制"一进去就被打"的体验问题，玩家始终先做决策
4. **预警机制** — 所有敌方强力招式都有前摇预警，感应技能可提前看到，鼓励策略而非蛮力
5. **BOSS 两阶段叙事战斗** — 锁血→觉醒的演出设计，战斗即叙事
6. **平台自适应** — VirtualJoystick 自动检测平台，PC 端零开销，Android 端显示触控 UI
7. **防御性编程** — 所有 await 后检查 is_inside_tree()，AutoLoad 信号在 _exit_tree() 断开
