# 有意义互动补全设计（TownScene v42 规划）

**作者决策日期**：2026-04-18（Opus 4.7 会话）
**执行模型建议**：Sonnet 4.6（方向已定，属于模式套用实现工作）

---

## 一、背景与问题

### 1.1 现状
- TownScene 中央 (x:400–700, y:400–550) 聚集了 **3 地标 + 3 剧情 NPC**（榕树/古井/公告栏 + 大婶/打水人/老狗），约 300×150 像素的小范围，显得拥挤
- 地图其他大片区域（左上、中右、下右、北路口）几乎完全空白
- 玩家在回程阶段走过中央会被连串叙事"拦住"，地图其他地方却缺少探索动力

### 1.2 用户要求
- RPG 需要更多**有意义的互动**，不是单纯点击即完成的空交互
- 允许加纯装饰物，但要让村子显得有"活人气息"
- 必须和现有剧情**圆得上**，不能瞎写

### 1.3 不做什么（硬性红线）
- **不动** `NPC_WaterCarrier` / `NPC_OldDog` / `NPC_AuntA` / `NPC_FortuneTeller` 的位置
  - 这 4 个 NPC 的坐标和 `TownScene.gd:66 FORCE_TRIGGER_NODES` 的 grid **一一对应**，是回程触发"挨个被叫住"节奏的设计锚点
  - 改动它们需要同时修 grid 表，且破坏"热闹→沉寂"的叙事节奏
- **不动** `NPC_Examiner`（广场在叙事上就应是聚集点）
- **不动** 已有的隐藏交互（`remnant_page_2` 老榕树、`remnant_page_3` 古井、`transmission_array` 传音阵）——它们已经在树洞/石缝/角落等合理位置
- **不重写**公告栏内容（`notice_board_before` / `notice_board_after` 已有完整双 phase 文本，主线钉子"无灵根者不在招募之列"不可动）

## 二、核心框架：「万事皆心石」

### 2.1 复用已有机制
不新建一套系统。直接复用 `TownScene.gd:88 HIDDEN_INTERACT_NODES` 模式：

```gdscript
{
    "grid":     Vector2i(x, y),
    "scene_id": "new_scene_id",
    "hint":     "按 E 查看",
},
```

走近自动出现提示气泡，按 E 触发 DialogueManager 场景，一次性读完即完成（由 `is_triggered` 守卫）。

### 2.2 心石计数绑定（新行为）
在新增交互点的 `set_flag` 字段里**同时**写入 `stones_read` 的增量标志（或直接在对话结束 hook 里 +1）。具体实现方式：

- **首选**：新增互动结束节点设 `set_flag: {"interaction_X_read": true}`
- 在 GameData 或 UIManager 监听这些 flag，首次触发时 `stones_read += 1`
- 读过的标识持久化，重读不再加

> 实现时参考已有的 `lore_page_X_read` 写法（chapter1.json:2831 等），保持一致

### 2.3 为什么按 E 而不是自动气泡
- 内容长度：每个互动是结构化观察/回忆（2–5 节），气泡承载不了
- 动作契合：读告示/观察石板/看物件，本质都是**主动行为**
- 情绪保护：phase 3+ 的变化是情绪拐点，需要"走到→按键→看见"三步小仪式
- 气泡机制保留给路人碎语（`BUBBLE_NODES`）使用

### 3.1 新互动文风约束
- 残页主题是"**他人笔迹的哲学碎片**"——克制、陌生、像拾到他人书页
- 新互动应是"**云晚视角的观察/回忆**"——第三人称限定、日常细节、不说透情绪
- 不写大段独白，不堆"感慨"类形容词
- Phase 3+ 的变化靠**环境或行为**暗示（被撕的告示、熄灭的灯、空着的位置），不靠多写一段心理描写
- 句子短。标点克制。参考现有 chapter1.json 的长句节奏

## 四、新增互动点方案（3 个高质量，不堆数量）

### ① 左上巷口·针线簸箩 `grid(8, 8)` = `pixel(256, 256)`

**外观**：竹凳一张 + 簸箩里放着一件修了一半的童装和一根针，墙角晾衣绳垂着两件粗布衣服。

**绑定主题**：邻里/童年——云晚小时候也穿过这样的衣服，巷口的婶子们修修补补长大的

**Phase 1 文案草稿**（3 节）：
```
rp5_01: 竹凳上搁着一只簸箩，针线歪在衣角上，像是有人刚走开。
rp5_02: 小时候她也穿过这样的衣服——袖口总被改短，因为长得太快。
rp5_03: 太阳晒着绳上的衣服，有一件在风里轻轻鼓起来，又落下去。
        set_flag: { neighborhood_memory_read: true }
```

**Phase 3+ 文案草稿**：
```
rp5a_01: 竹凳还在，簸箩空了。衣服收走了。
rp5a_02: 她想起那件袖口改短的旧衣服——不知道后来去了哪里。
        set_flag: { neighborhood_memory_read_after: true }
```

### ② 北路口·测灵广场路标 `grid(22, 6)` = `pixel(704, 192)`

**外观**：一块顶端带斜切的木牌钉在石桩上，"测灵广场 →" 四字漆色斑驳，箭头指向东北。

**绑定主题**：测灵焦虑具象化——牌子越走越近，心越走越沉；回程时这块牌子就是反向的伤口

**Phase 1 文案草稿**：
```
sp1_01: 一块木牌钉在石桩上。
sp1_02: 「测灵广场 →」四个字，漆色已经剥得差不多了，箭头还认得出来。
sp1_03: 她顺着箭头的方向看了一眼——人影已经聚起来了。
        set_flag: { signpost_read: true }
```

**Phase 3+ 文案草稿**：
```
sp1a_01: 箭头还指着那边。
sp1a_02: 她没有再顺着看过去。
        set_flag: { signpost_read_after: true }
```

> 注：文字极简是刻意的。对比 phase 1 的 3 节和 phase 3+ 的 2 节——"少一节"本身就是情绪。

---

### ③ 东墙根·磨平的石板 `grid(28, 16)` = `pixel(896, 512)`

**外观**：一块半埋在墙根的方石，表面被日头晒得发白，隐约有刻痕但已辨认不清；旁边长着几丛野草。

**绑定主题**：遗忘/时间——"什么都会过去"的冷色调，和残页的"不忘"形成对照。放在远离中央的墙角，让路过的玩家自然遇到

**Phase 1 文案草稿**：
```
sl1_01: 墙根埋着半块方石，晒得发白。
sl1_02: 她蹲下来看了看——石面上有刻痕，像是字，但已经辨认不清。
sl1_03: 刻这行字的人大概也没想到，这字会有一天谁都看不懂。
        set_flag: { worn_stone_read: true }
```

**Phase 3+ 文案草稿**：
```
sl1a_01: 那块石头还在。
sl1a_02: 她忽然想——被测灵石测过的手印，过几年是不是也会被风磨平？
sl1a_03: 她没有给自己回答。
        set_flag: { worn_stone_read_after: true }
```

## 五、装饰物清单（无交互，视觉 only）

目的：让村子显得"有人在生活"，分散空白区的空虚感。全部走 `_spawn_world_decorations()` 里现有的 Polygon2D 模式，**不需要**碰撞体、不需要 Area2D。

| # | 位置 | 内容 | 实现提示 |
|---|---|---|---|
| D1 | `(288, 320)` 左上巷 | 晾衣绳（两点间一条 Polygon2D 细线）+ 两块色块代表衣服 | 粗布色 `Color(0.55, 0.50, 0.42)` / `Color(0.72, 0.50, 0.38)` |
| D2 | `(192, 608)` 市场西 | 墙根一丛野花（3–5 个小椭圆，不同色） | 暖色堆叠 |
| D3 | `(480, 736)` 市场中 | 一只走动的鸡（小椭圆身 + 尖喙三角 + 短腿），静态即可 | 米白 + 红冠点缀 |
| D4 | `(832, 736)` 东南民居檐下 | 屋檐下垂着的瓦罐（两个小梯形，色差叠加） | 陶土色 |

## 六、实现顺序建议

1. **Step 1**（最低成本见效）：先做 3 个装饰物（D1 + D2 + D5）——因为它们在地图最空的地方。不到 80 行代码。
2. **Step 2**：新增互动点 ② 路标——最简单的一个，phase 变化也最短。跑通整条链路（HIDDEN_INTERACT_NODES 注册 + JSON 场景 + set_flag + stones_read 绑定）。
3. **Step 3**：新增互动点 ①、③——复制 Step 2 的模式。
4. **Step 4**：补完剩余装饰物 D3、D4、D6。
5. **Step 5**（可选）：检查 UIManager 心绪面板在 `stones_read` 增加时是否自然刷新文字；如不刷新，评估是否让这些互动有"心绪 tick"。
